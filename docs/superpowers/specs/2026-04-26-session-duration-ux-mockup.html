Reading the mockup

  1. Sidebar stays as-is. The active session shows "live" instead of an ago time.
  2. Conversation — every assistant message gets a turn-summary chip in its header (e.g. turn 2m 18s · 3 calls). Slow turns get the rose tint. Running turn pulses in green family.
  3. ParallelGroup — assistant turn 2 has 3 parallel calls, wrapped in a solid neutral rail with header parallel · 3 calls · ≤2m 14s each. Inside, the Read tool blocks lose their per-call duration badge (the group owns it); the Agent sub-agent keeps its exact duration since we know it from the child session.
  4. Sub-agent inline — the Agent in turn 2 is expanded; child calls show with their own duration badges (9ms, ≤11ms for parallel reads inside, 2m 8s for the slow Bash). The same hierarchy is browsable via the chevron on the Agent row in the right panel.
  5. Right panel sections, top to bottom:
    • Session summary — running time pulses green; slowest call excludes the in-flight call; in flight tile shows the current running tool.
    • Time spent — bars sum to total completed tool wall-clock; running turn doesn't yet contribute (its category is undecidable until it ends).
    • Timeline lanes — turns lane shows each turn's duration colored by primary category; per-category lanes show the same data sliced; activity lane keeps the existing message-density chart. Running turn shows a pulsing green segment.
    • Calls — chronological list; ParallelGroup uses a left-rail container with the same encoding as the conversation; sub-agents expand via chevron with full nested hierarchy; the running call has a green pulsing bar; scale axis ends at now.
